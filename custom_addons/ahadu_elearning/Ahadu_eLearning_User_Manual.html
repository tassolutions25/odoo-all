1. Introduction

1.1 About Ahadu eLearning

The Ahadu eLearning module is a customized extension of Odoo 18's eLearning (Website Slides) platform, specifically tailored for Ahadu Bank S.C.. It provides a comprehensive Learning Management System (LMS) that enables the bank to create, manage, and deliver training courses and educational content to employees across all branches.

This module empowers the bank's Human Resource Development and Training department to centralize all learning activities, track employee progress, and ensure compliance with banking industry training requirements.

1.2 Key Features

FeatureDescription
Course ManagementCreate and organize training courses with sections, categories, and tags
Multi-format ContentSupport for videos, documents, infographics, web pages, and articles
Local Video UploadUpload video files directly (up to 1.2 GB) from your device
Google Drive IntegrationEmbed videos from Google Drive with automatic player rendering
External Link SupportLink to external training resources and video platforms
Quiz & AssessmentsCreate quiz questions with multiple answers for knowledge evaluation
Course TaggingOrganize courses using tags and tag groups for easy filtering
Progress TrackingMonitor employee enrollment and completion status
Ahadu BrandingComplete Ahadu Bank branding throughout the interface

1.3 System Requirements

  • Odoo 18 Community or Enterprise Edition
  • Website Slides (website_slides) module installed
  • Modern web browser (Chrome, Firefox, Edge, or Safari)
  • Stable internet connection

1.4 User Roles

RoleAccess LevelDescription

eLearning ManagerManager AccessCreate and manage courses, view reports and analytics.
eLearning OfficerUser AccessCreate content, manage assigned courses.
Employee (Learner)Portal AccessEnroll in courses, view content, take quizzes.

3.4 Editing and Deleting Courses

To Edit: Click on any course from the list to open it. Modify the desired fields and save.

To Delete: Open the course, click the ⚙️ Actions (gear icon) menu, and select "Delete". Confirm the deletion when prompted.

💡 Tip: Deleting a course will also remove all its content (slides, quizzes). Consider archiving instead by setting the course to unpublished.

6.3 Quiz Best Practices for Banking Training

💡 Tip: For compliance-related courses, include at least 3-5 quiz questions per section to ensure proper knowledge retention.
  • Use clear, unambiguous question wording
  • Provide 3-4 answer options for each question
  • Include feedback comments for wrong answers to guide learning
  • Mix question types: factual, scenario-based, and application-based
  • Ensure questions align with Ahadu Bank policies and procedures

8. Best Practices & Tips

8.1 Course Design Best Practices

  • Keep content modular — Break courses into small, focused sections (5-15 minutes each)
  • Mix content types — Combine videos, documents, and articles for engagement
  • Use sections — Group related content into logical sections for better navigation
  • Add preview content — Mark introductory slides as "Preview" so employees can see what the course covers before enrolling
  • Include quizzes — Add quiz questions after each section for knowledge reinforcement

8.2 Video Upload Guidelines

AspectRecommendation
File FormatMP4 (H.264 codec) for best compatibility
Maximum Size1.2 GB per video file
Resolution720p (1280×720) for training videos
Duration5-20 minutes per video segment
CompressionUse CRF 23-28 in HandBrake for good quality-size balance
ℹ️ Note: For Google Drive videos, ensure the sharing setting is set to 'Anyone with the link can view' so all enrolled employees can access the content.

8.3 Troubleshooting

IssueSolution
Video not playingCheck file format (use MP4). For Drive videos, verify sharing settings.
Upload failsEnsure file size is under 1.2 GB. Check internet connection stability.
Course not visible to employeesVerify the course is "Published" and enrollment policy is correct.
Quiz not showingEnsure the parent slide is published and quiz questions have correct answers marked.
Tags not appearingConfirm tags are saved and refresh the page.
